Prune it, fathom it, or keep branching? Slides 6–9

Implicit enumeration lives on three elimination rules. Each card below puts you at one node of a 0-1 tree with some variables fixed — decide what happens to the node. All three instances are the slides’ own.

Rule 1 — best completion is feasible

Max z = 4x₁ + 2x₂ − x₃ + 2x₄
s.t. x₁ + 3x₂ − x₃ − 2x₄ ≥ 1,  x ∈ {0,1}⁴
Node: x₁ = 0, x₂ = 1 fixed. Pick the completion (x₃, x₄) that the rule would choose.

Best values for the free variables = ?

Rule 2 — bound against the incumbent

Max z = 4x₁ + 2x₂ + x₃ − x₄ + 2x₅,  x ∈ {0,1}⁵
Current best integer solution: z = 6.
Node: x₁ = 0, x₂ = 1, x₃ = 1 fixed. What z does the best completion of this node reach?

Free variables: x₄ and x₅.

Rule 3 — no feasible completion at all

Constraint: −2x₁ + 3x₂ + 2x₃ − 3x₄ − x₅ + 2x₆ ≤ −5,  x ∈ {0,1}⁶
Node: x₂ = 1, x₃ = 1, x₄ = 1 fixed. How small can the left-hand side possibly get?

Free variables: x₁, x₅, x₆ — push the LHS down as hard as you can.

These are exactly the three moves the algorithm makes at every node: fathom by a feasible best completion (Rule 1), prune by bound (Rule 2), prune by infeasibility (Rule 3).
spoiler — open after the class has answered Rule 1: the objective-best completion is x₃ = 0, x₄ = 1 → z = 4, and it satisfies the constraint (LHS = 1 ≥ 1), so the node is solved — no branching. Rule 2: best completion x₄ = 0, x₅ = 1 gives z = 2 + 1 + 2 = 5 < 6, so the node dies even though we never checked feasibility. Rule 3: the most helpful free values (x₁ = 1, x₅ = 1, x₆ = 0) still leave LHS = −1 > −5 — no completion can ever be feasible, prune.

Walk the implicit-enumeration tree Slides 10–13

The slides’ Example 1: max z = −7x₁ − 3x₂ − 2x₃ − x₄ − 2x₅  s.t.  −4x₁ − 2x₂ + x₃ − 2x₄ − x₅ ≤ −3  and  −4x₁ − 2x₂ − 4x₃ + x₄ + 2x₅ ≤ −7, x ∈ {0,1}⁵. You call every move; the tree grows as you answer. 2⁵ = 32 leaves — how few nodes can kill them all?

1. Branch on x₁ first. Which branch can be eliminated on the spot?
2. At the x₁ = 1 node, branch on x₃. Which branch dies now?
3. At the x₁ = 1, x₃ = 1 node — best completion (x₂, x₄, x₅)?

Hint: for each branch, give every free variable its most constraint-friendly value and test constraint 2.

Three questions, five nodes drawn, and all 32 assignments are accounted for — that is the “implicit” in implicit enumeration.
spoiler — open after the class has answered x₁ = 0 dies (constraint 2 bottoms out at −6 > −7), then x₃ = 0 dies (bottoms out at −2 > −3), and at x₁ = x₃ = 1 the objective-best completion x₂ = x₄ = x₅ = 0 is feasible: fathomed with z = −9, the optimum — exactly the slides’ path.

Beat the enumeration — find the best 0-1 point Slides 14–15

Example 2 from the slides: max z = 3x₁ + x₂ + 2x₃ − x₄ + x₅  s.t.  2x₁ + x₂ − 3x₄ ≤ 1  and  x₁ + 2x₂ − 3x₃ − x₄ + 2x₅ ≥ 2, x ∈ {0,1}⁵. Toggle the five variables, watch both constraints, and log your best feasible z before the algorithm reveals its answer.

2x₁ + x₂ − 3x₄ = 0 (need ≤ 1) x₁ + 2x₂ − 3x₃ − x₄ + 2x₅ = 0 (need ≥ 2) z = 0

All zeros: constraint 2 gives 0 < 2 — infeasible. Start toggling.

The cutting-plane machine — from tableau to cut to pivot Slides 25–32

Same example, now in the tableau. Start from the optimal tableau of the LP relaxation (slide 25) and run the algorithm: click the row that should generate the cut, derive it, add it, choose the entering variable, pivot — and repeat until every value is integer.

current LP solution: x₁ = 2, x₂ = 5/2, z = 9/2

Source row: \( x_2 + \tfrac{1}{6}y_1 + \tfrac{1}{6}y_2 = \tfrac{5}{2} \). Keep the fractional pieces on the left, move the integer pieces to the right:

\[ \tfrac{1}{6}y_1 + \tfrac{1}{6}y_2 \;=\; \tfrac{1}{2} + (2 - x_2) \]

At any integer solution the right side is \( \tfrac12 \) plus an integer, and the left side is \( \ge 0 \) — so the left side can never drop below \( \tfrac12 \). The cut (equivalent to \( x_2 \le 2 \), added with surplus \( y_3 \)):

\[ \tfrac{1}{6}y_1 + \tfrac{1}{6}y_2 \;\ge\; \tfrac{1}{2} \]

Source row: \( x_1 + \tfrac{1}{5}y_2 - \tfrac{2}{5}y_3 = \tfrac{11}{5} \). Add \( y_3 \) to both sides so every fraction is positive \( (-\tfrac25 = -1 + \tfrac35) \), then split integers from fractions:

\[ \tfrac{1}{5}y_2 + \tfrac{3}{5}y_3 \;=\; \tfrac{1}{5} + (2 - x_1 + y_3) \]

Same argument: the left side can never drop below \( \tfrac15 \). The cut (in \( (x_1, x_2) \)-space: \( x_1 + x_2 \le 4 \), added with surplus \( y_4 \)):

\[ \tfrac{1}{5}y_2 + \tfrac{3}{5}y_3 \;\ge\; \tfrac{1}{5} \]

Source row: x₂ + (7/22)s₁ + (1/22)s₂ = 7/2. Split every number into integer + fraction, then decide what the fractional side may do.

A valid cut must do two jobs at once: keep every feasible integer point, and throw out the current fractional optimum. Test each candidate on both jobs.
spoiler — open after the class has answered C is the Gomory cut (slide 35): x₂ − 3 = ½ − (7/22)s₁ − (1/22)s₂ has an integer left side, and the right side — a fraction minus nonnegatives — can only be 0, −1, −2, …, never positive. At the current optimum s₁ = s₂ = 0 the left side is ½ > 0, so the point (9/2, 7/2) is cut off (slide 36). In (x₁, x₂)-space the cut reads x₂ ≤ 3.
