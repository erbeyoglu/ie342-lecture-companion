Which investments fit $24,000? Slide 4

The investor on slide 4 has $24,000 and four opportunities (costs $9k, $7k, $7k, $3k; present values $13k, $11k, $10k, $5k). Click cards to build a portfolio — every investment is all-or-nothing, that is what makes the x₁…x₄ binary. Hunt the largest total present value that still fits the budget.

Modeling gate — say it in binary. Before the hunt: the investor also has three house rules, and each is a single linear constraint on x₁…x₄. Model all three and the cards unlock. (Quiz items written for this demo; slide 4 states only the budget model.)

A quick trap to try on the class first: ranking by “bang per buck” ($5k/$3k = 1.67 is the juiciest ratio) and filling greedily gives portfolio {1, 2, 4} — is that the best this budget can do?
spoiler — open after the class has answered The optimum is {1, 2, 3}: cost $23,000 ≤ $24,000, present value $34,000. Greedy-by-ratio reaches only $29,000 because taking cheap investment 4 blocks investment 3. Cover the city with fire stations Slides 13–16

The city map from slide 13: eleven neighborhoods, and a station handles its own neighborhood plus every neighborhood sharing a border with it. Click neighborhoods to open (or close) stations — covered areas fill in. Protect all 11 with as few stations as you can.

Modeling gate — build the coverage row. Slide 14 writes one row per neighborhood: xj enters neighborhood i's row exactly when a station at j would protect i. Click, on the map, every location that would protect neighborhood 11, then check. Build two rows and the hunt unlocks.

spoiler — open after the class has answered The minimum is 3 stations — no pair covers everything (checked by enumeration on the slide's adjacency data). Which constraints are redundant? Slides 14–16

Slide 14 writes one covering constraint per neighborhood and then asks the class: which of these eleven are redundant? Click rows to strike the ones you would delete, then check. A constraint is safe to delete only if some other row forces it automatically.

struck: 0

The rule of thumb: if row A's variable set contains row B's entirely, then satisfying B (≥ 1) automatically satisfies A — so A can go.
spoiler — open after the class has answered Strike rows 3, 6, 9 and 10, exactly as slide 15 does: row 3 contains row 1 (and row 2), row 6 contains row 7, and rows 9 and 10 both contain row 11. Slide 16 keeps the remaining seven constraints. Deleting redundant rows never changes the feasible set — it just makes the model smaller.
