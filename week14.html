Write the flow-balance rows Slides 3–5

The week's first example: two supply points (4 and 5 units at nodes 1–2) ship to two demand points (6 and 3 units at nodes 3–4) along four arcs with unit costs 1, 2, 3, 4. Every MCNFP constraint says net flow out of node i = bi. Build that constraint matrix: click each cell to cycle its coefficient through 0 → +1 → −1, type the right-hand sides, then check.

All coefficients start at 0.

Generic MCNFP (slide 3): min Σ cijXij subject to ΣjXij − ΣkXki = bi at every node and Lij ≤ Xij ≤ Uij on every arc. Transportation, assignment, transshipment, shortest path, max flow and CPM are all special cases.
spoiler — open after the class has answered Slide 5's model: min Z = X₁₃ + 2X₁₄ + 3X₂₃ + 4X₂₄ with rows X₁₃ + X₁₄ = 4 (node 1), X₂₃ + X₂₄ = 5 (node 2), −X₁₃ − X₂₃ = −6 (node 3), −X₁₄ − X₂₄ = −3 (node 4). Every column holds exactly one +1 and one −1: each unit of flow leaves one node and enters another. (For the curious: the cheapest such flow costs 22 — computed from the slide's data; the deck stops at the formulation.)

spoiler — open after the class has answered Total supply 45 + 120 + 95 = 260; total demand 80 + 78 + 47 + 55 + 25 = 285. The network is NOT balanced — slide 19's punchline — so an MCNFP needs a dummy plant with b = +25 whose zero-cost arcs absorb the unmet demand (the reveal draws one standard completion; the slide states the need without drawing it). Spanning-tree warm-up Slides 20–21

Before touching the examples, settle slide 20's and slide 21's questions. Answer all three — each wrong guess earns a targeted hint.

1 · A spanning tree on the 5-computer campus network uses how many arcs?

2 · Is the MST problem easy or hard to solve?

3 · Does Prim's greedy algorithm always end at a true MST?

Slide 20: a spanning tree on n nodes is a set of n − 1 arcs connecting all nodes with no loops; the shortest one is the MST.
spoiler — open after the class has answered 1: n − 1 = 4 arcs. 2: easy — unusually for combinatorial optimization, a greedy pass is exact. 3: always; ties for the closest node may be broken arbitrarily (slide 21) and every choice still ends in a minimum spanning tree.

Prim's algorithm — the class calls every arc Slides 22–26

The State University campus: five computers, cable distances on the links (two machines are unlinked — underground rock). Prim starts from node 1, as on slide 23. At every iteration, click the arc you think Prim adds next — the cheapest arc from the connected set C to an unconnected node. Wrong calls get corrected; ties are yours to break.

C = {1} length so far: 0

Iteration 1 — C = {1}. Click the arc Prim adds first.

This is exactly the loop of slide 21: pick the unconnected node closest to C, add the arc that realizes that distance, repeat until C holds everything.
spoiler — open after the class has answered Slides 23–26: arc (1,2), then the 2-block tie for node 5 — the slide picks (2,5) — then (5,3) and finally (5,4). The minimum spanning tree has length 1 + 2 + 2 + 4 = 9 blocks. Taking (1,5) at the tie gives a different tree with the same length 9.
