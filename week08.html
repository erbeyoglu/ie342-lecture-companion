Sack is empty — weight 0 / 14, z = 0. Slide items in and out.

A knapsack problem is an IP with a single constraint (slide 4). With 0–1 variables its LP relaxation needs no simplex: rank items by value per unit weight and pour greedily until capacity runs out.
spoiler — open after the class has answered Ratios: 8/5 = 1.6 > 11/7 ≈ 1.57 > 6/4 = 1.5 > 4/3 ≈ 1.33. Pouring in that order fills items 1 and 2 whole and dies half-way through item 3: LP optimum x = (1, 1, 0.5, 0) with zLP = 22 (slide 5) — so we branch on x₃. The best 0–1 packing is items 2, 3, 4 with z = 21; the branch-and-bound tree below proves no packing beats it.

The same knapsack, run as branch-and-bound exactly as in slides 5–11. Branching on a 0–1 variable just means an x = 0 child and an x = 1 child, and every LP bound is read off by inspection. Every node stores the LP bound of its subtree; a node is fathomed when its LP is integer, infeasible, or bounded below the incumbent.
spoiler — open after the class has answered Optimal: z = 21 at x = (0, 1, 1, 1). Slide 11’s question — must we branch the x₃ = 0 node (bound 21.65)? No: all objective coefficients are integers, so the best integer value under it is at most ⌊21.65⌋ = 21, which cannot beat the incumbent 21. Fathomed by bound, subtree never built.

Cheaper than any tour: build the assignment Slides 14–16

Slides 14–15 relax the tour: from each city Joe must leave exactly once and each city must be entered exactly once — with no promise that the arcs form one loop. That is an assignment problem (diagonal forbidden: cᵢᵢ = M). Click one cell in every row and column of the cost matrix and watch the arcs on the map. Can the class build an assignment cheaper than its best tour from the widget above? What goes “wrong” with the cheapest ones?

arcs picked = 0 / 5 cost = 0

Pick a successor in every row — one cell per row and per column.

Every tour is a legal assignment, but not every assignment is a tour — so the optimal assignment value is a lower bound on the optimal tour. And the subproblems here are not LP relaxations (slide 14) — branch-and-bound only needs some relaxation it can solve fast.
spoiler — open after the class has answered Hungarian optimum (slide 16): x₁₅ = x₅₂ = x₂₁ = 1 and x₃₄ = x₄₃ = 1, cost 495 — two subtours, GA → SB → FW → GA and EV → TH → EV, so 495 ≤ 668 is a bound, not a route. Slide 16 branches to destroy EV ↔ TH: subproblem 2 forbids EV → TH (c₃₄ = M), subproblem 3 forbids TH → EV (c₄₃ = M).

Branch and bound over assignments — kill the subtours Slides 16–21

Slides 16–21 run B&B where every node is an assignment problem solved by the Hungarian method. Step through the whole search: spot subtours, pick the branching, and decide what may be pruned — the widget stops and asks the class at each judgement call. Click a solved node in the tree to see its assignment on the map.

SP1 relaxes the tour requirement to a pure assignment — solve it first.

Branching rule (slide 16): pick a subtour and create one child per arc of it, each child forbidding one arc (cost set to M) — the subtour survives in no child, yet every genuine tour survives in at least one.
spoiler — open after the class has answered Search ends as on slide 21: SP4’s single-loop solution GA → SB → FW → TH → EV → GA with z = 668 is optimal. SP5 is a 704 tour (eliminated), and SP6 / SP7 — the children of SP3 (bound 652) — cannot produce anything below 668, so both are pruned.
